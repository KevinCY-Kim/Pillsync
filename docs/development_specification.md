# PillSync - 맞춤형 영양제 매칭 플랫폼 개발 명세서

본 문서는 **PillSync(필싱크)** 플랫폼의 데이터베이스 아키텍처, UI/UX 이원화 설계 및 개발 과정에서 합의되고 구현된 사양들을 정리한 종합 개발 명세서입니다.

---

## 1. 프로젝트 개요
PillSync는 대한민국 식품의약품안전처(MFDS)의 건강기능식품 고시 정보를 바탕으로, 사용자 개개인의 건강 고민(피로, 다이어트, 탈모 등)을 분석하여 최적의 영양소 성분 조합과 시너지를 제공하고, 쿠팡 파트너스 API 최저가 딥링크를 연결해 즉각적인 구매 전환을 돕는 **개인 맞춤형 웰니스 큐레이션 서비스**입니다.

---

## 2. 시스템 아키텍처 & DB 스키마 구조

플랫폼은 클라우드 DB 모드(Supabase)와 클라이언트 오프라인 인메모리 DB 모드(Fallback)를 이중 지원하여 높은 안정성과 비연결 상태의 가용성을 보장합니다.

```
                  ┌──────────────────────┐
                  │   Vite React App     │
                  └──────────┬───────────┘
                             │
            ┌────────────────┴────────────────┐
     [Supabase Configured?]            [Offline Fallback]
            ▼                                 ▼
┌──────────────────────┐           ┌──────────────────────┐
│  Supabase Cloud DB   │           │ Local Mock In-Memory │
└──────────────────────┘           └──────────────────────┘
```

### 🗂️ 테이블 스키마 설계 및 무결성 제약 조건
*   **`categories` (대분류 테이블)**: 건강 고민 분류 (피로 개선, 다이어트, 탈모 등).
*   **`ingredients_mapping` (식약처 영양소 매칭)**: 식약처 기능성 요약, 일일 권장량, 고함량 기준 메리트, 주의해야 할 부작용 기저 질환, 그리고 대안 성분 ID(자기 참조 외래키) 및 대안 추천 사유 적재.
*   **`symptoms` (설문지 선택 항목)**: 사용자가 선택할 체크박스 문구와 각 카테고리/매칭 성분 간의 외래키 연동.
*   **`synergy_combinations` (시너지 조합)**: 영양소들 간의 병용 섭취 효과와 추천 사유 명세.
*   **`synergy_ingredients` (시너지 매핑 브릿지)**: 다대다 관계 해소를 위한 중간 테이블.

> [!TIP]
> 상세 스키마 선언문 및 초기 데이터 삽입 스크립트는 프로젝트 루트의 [supabase_schema.sql](file:///c:/Users/stone/projects/PillSync/supabase_schema.sql) 파일에서 통합 관리하고 있습니다.

## 3. 웹 & 모바일 접속 환경 이원화 최적화 명세

다양한 디바이스와 데모 목적을 충족하기 위해 프론트엔드 뷰(View)를 스마트하게 이원화했습니다.

### 📱 1) 모바일 기기 접속 최적화 (Mobile-First Layout)
*   **앱 디자인 고도화**: 스마트폰 액자 형태의 테두리(`.mobile-frame`) 및 데스크톱 전용 헤더/푸터 데코레이션을 완전히 걷어내어 모바일 브라우저 전체 너비와 높이를 100% 가득 메우도록 스타일링을 오버라이딩했습니다.
*   **메모리 및 렌더링 효율**: 모바일 기기 감지 시 어드민 제어판(`.admin-section`)은 DOM 트리에서 완전히 제거되어 로딩 병목 현상과 과도한 CPU 점유를 예방합니다.

### 💻 2) 데스크톱 웹 접속 최적화 (Dual-Mode Toggle)
*   **일반 모드**: 일반 웹 브라우저 진입 시 중앙에 에뮬레이션 폰 가이드만 큼직하게 정렬하여 프로토타입 앱의 본질에 집중하게 만듭니다. 일반 유저에게는 헤더 내 데모 토글 버튼 자체가 아예 보이지 않습니다.
*   **데모 관리자 모드**: 관리자나 바이어가 비밀 쿼리 파라미터가 포함된 특수 주소(예: `.../?admin=true`)로 접근할 경우에만, 헤더 우측 상단에 **`⚙️ 데모 관리자 패널`** 토글 스위치가 활성화되고 켜진 채로 로드됩니다. 이를 통해 데이터 입력 제어판 및 DB 스키마 뷰어가 노출됩니다.

## 4. 로컬 구동 및 배포 가이드

### 🚀 로컬 개발 서버 기동
1. 의존성 설치: `npm install`
2. 개발 서버 실행: `npm run dev`
3. 로컬 빌드 및 오류 검증: `npm run build`

### 🌐 Vercel 호스팅 연동
*   본 프로젝트는 GitHub `main` 브랜치와 Vercel 배포 파이프라인이 자동 바인딩되어 있습니다. `git push origin main` 명령 시 약 1분 이내에 [pillsync-ten.vercel.app](https://pillsync-ten.vercel.app/)으로 실시간 배포가 완료됩니다.
