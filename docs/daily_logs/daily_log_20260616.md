# PillSync Daily Development Log - 2026.06.16

## 1. 작업 개요
오늘 진행된 작업은 PillSync 서비스의 PWA 기능 고도화, 신규 건강 고민 카테고리(눈 건강, 혈행 개선, 수면 & 스트레스) 탑재 및 쿠팡 파트너스 수익화 연동, 의료법 및 개인정보보호 관련 법적 컴플라이언스 준수 강화, 데스크톱 뷰 레이아웃과 아코디언 등 UI/UX 편의성 개선, 그리고 페이지 로딩 시 글자가 변하는 현상을 잡기 위해 로컬 Mock DB와 Supabase 클라우드 DB의 데이터 정합성을 일치시키는 데 중점을 두었습니다.

---

## 2. 상세 작업 내역

### 1) PWA 지원 고도화 및 모바일 설치 흐름 최적화
* **Play Protect 경고 회피**: 모바일 기기(삼성 인터넷 등)에서 원클릭 PWA 설치를 강제할 경우 보안 수준에 따라 차단 경고가 뜨는 문제를 방지하기 위해, 모바일 환경에서는 PWA 홈 화면 즐겨찾기(바로가기) 추가 가이드 모달로 안전하게 우회하도록 처리.
* **PWA 구성 요소 보강**: 브라우저 수준에서의 무결성을 유지하고 보안 경고를 예방하기 위해 고유 크기(192px, 512px)의 PWA 아이콘 생성 및 사전 캐싱 레이어를 구축.
* **설치 권장 배지 숨김 로직**: 홈 화면 추가 권장 배지를 사용자가 직접 끄거나 30일 동안 노출을 일시 차단할 수 있는 만료 기한 연동 로직 구현.
* **가이드 가독성 향상**: 삼성 인터넷의 '현재 페이지 추가' 메뉴를 활용한 북마크 등록 등 브라우저 환경에 맞는 PWA 안내 가이드 내용 상세화.

### 2) 3대 신규 건강 고민 카테고리 탑재 및 수익화 연동
* **카테고리 확장**: 기존 3개 카테고리에 더해 `눈 건강`, `혈행 개선`, `수면 & 스트레스` 3개 카테고리를 추가하여 총 6개 카테고리(식약처 공인 15개 기능성 성분 데이터) 구조로 확장.
* **수익화 연동**: 신규 3개 카테고리에 매핑되는 성분(헤마토코쿠스, 루테인지아잔틴, 오메가3, 은행잎, 감태, 락티움 등)에 적합한 실제 쿠팡 파트너스 딥링크 및 모의 상품 데이터 매핑.
* **기능 보완**: 식약처 고시 데이터 조회 모달(`KfdaReportModal`)에 새로운 카테고리와 성분 규격(`kfdaData`)을 수동 탑재하고, Supabase 연동 시의 대분류 아이콘 매핑 폴백 클래스(`cat-blood`, `cat-sleep` 등)가 정상 부여되도록 완료.

### 3) 로컬 Mock DB와 Supabase 데이터 싱크 일치 (단일 진실 공급원)
* **텍스트 튐 버그 해결**: 페이지 새로고침 시 로컬 오프라인 데이터와 Supabase 클라우드 데이터가 미묘하게 달라 렌더링 시 화면 텍스트가 깜빡이며 변하는 문제를 해결하기 위해, 로컬 모크 데이터 구조와 Supabase 시드 SQL 데이터를 일대일로 완벽하게 동기화.
* **SQL 스크립트 멱등성 및 정합성 보완**: 
  - 중복 기재로 DB 적재 오류를 일으켰던 중복 행(예: `헤마토코쿠스 추출물` 중복 행)을 완전히 정돈.
  - 몇 번을 재실행해도 데이터가 깨지거나 에러를 내지 않도록 `ON CONFLICT` 기반 UPSERT 로직 및 Serial 시퀀스 자동 초기화 구문 보완.

### 4) 법적 컴플라이언스 및 의료법 위반 리스크 대응
* **표현 정제**: 식약처 사칭이나 진단 오인 리스크를 예방하기 위해 원격 DB 연동 시 표기되던 규제 위반 문구들을 로컬의 정제된 컴플라이언스 매핑 텍스트로 오버라이드.
* **면책 고지 및 개인정보 고지**: 설문 결과 페이지 하단 등에 개인 건강정보는 서버에 전혀 수집·전송되지 않고 브라우저에만 유지된다는 고지와 예시용 쿠팡 상품 안내 문구 등 배치.
* **ESLint 및 React Hook 에러 수정**: ESLint 정적 에러와 PWA 안내 팝업 관련 훅 에러 등 코드 안정성 저해 요소 제거.

### 5) UX/UI 편의성 개선 및 폴리싱
* **데스크톱 레이아웃 개선**: 카테고리 카드가 지나치게 넓게 퍼지지 않고 모바일 뷰와 일맥상통하도록 데스크톱 뷰 카드 세로 정렬(Stack) 레이아웃으로 변경.
* **관리자 패널 스타일 버그 수정**: 관리자 패널 내 입력 필드와 컴포넌트의 수직 정렬 어긋남 해결.
* **섭취 정보 가독성 증대**: 사용자가 별도로 클릭하지 않아도 부작용 가이드를 눈에 띄게 확인할 수 있게 결과 화면의 아코디언 가이드를 기본 펼침(`default-expand`) 상태로 전환.
* **컴포넌트 메모리 상태 초기화**: 카테고리를 리셋하거나 다른 건강 카테고리로 변경 시, 이전 화면의 주의 문구 체크 정보(`checkedWarnings`) 및 열려 있는 아코디언 상태(`openGuides`)도 자동 리셋되도록 보완.
* **타이포그래피 및 폰트 줄바꿈**: 브랜드명이 디바이스 가로 폭에 따라 보기 싫게 끊어지지 않도록 강제 줄바꿈 및 간격 보완.

### 6) 분석 도구 통합
* **Vercel Analytics 연동**: React Entry point 파일에 Vercel Analytics 패키지를 통합하여 사용자 트래픽 통계 및 방문자 수치 추적 기반 마련.

## 3. 빌드 및 배포 상태

* **로컬 컴파일 테스트**: `npm run build` 프로덕션 빌드 결과, 어떠한 컴파일 경고나 컴파일 에러 없이 `dist/` 빌드가 성공적으로 완수되었습니다.
* **원격 DB 배포 및 연동**: 수정한 시드 데이터가 Supabase와 완벽히 호환되도록 구성되었으며, 로컬 로딩 시 텍스트 깨짐이나 튀는 현상 없이 깔끔하게 동작합니다.
