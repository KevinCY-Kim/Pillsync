## 2. 상세 작업 내역

### 1) 보안 하드닝 (Security Hardening)
* **[🔴 S1] Supabase DB 공개 쓰기 제한 및 Auth 인증 도입**
  - 관리자 패널(`?admin=true`)에 진입 시 로그인 세션을 확인하도록 로그인 폼 및 로그아웃 바를 구현했습니다.
  - 데이터베이스의 쓰기 권한(`INSERT`, `UPDATE`, `DELETE`)을 비인증 사용자(`public`)로부터 격리하여, 인증된 사용자(`authenticated`) 전용 정책으로 Supabase RLS 정책을 변경했습니다.
  - 로컬 오프라인 모드에서는 회원 인증 없이도 관리자 편집 폼이 동작하도록 로직(`canUseAdminWrites`)을 예외 처리했습니다.
* **[🔴 S2] click_logs 공개 읽기 권한 박탈**
  - 민감할 수 있는 사용자 접속 로그(`click_logs`)의 전역 조회 권한을 제한하여 익명 사용자는 조회할 수 없도록 수정했습니다.
* **[🟡 R1] PWA 오프라인 구동 시 빈 화면 버그 패치**
  - 서비스 워커(`sw.js`)에서 동일 출처의 `GET` 요청 결과(해시된 정적 자산들)에 대해 런타임 캐싱을 수행하도록 기능을 보강하여, 오프라인 환경에서도 정상적으로 PWA 셸과 번들이 로드되도록 개선했습니다.
* **[🟡 R2] 로컬 모드 데이터 ID 충돌 예외 방지**
  - 관리자 폼으로 새로운 카테고리를 등록할 때 단순 길이 기준(`length + 1`)이 아닌 최댓값 기준(`MAX(id) + 1`)으로 자동 부여하여 기존 시드 예약 ID와 충돌하지 않도록 방지했습니다.

### 2) 유튜브 쇼츠 인앱 브라우저 우회 연동
* **인앱 브라우저 타겟 우회**: YouTube 및 Twitter 모바일 인앱 브라우저(WebView)에서 쿠팡 카드 클릭 시 새 탭(`_blank`)이 막히는 문제를 감지하여, 인앱 웹뷰 상태일 때 현재 창 리다이렉트(`window.location.href`)로 변경해 유입 클릭 유실을 해결했습니다.
