### 2) 서비스UX 개선 및 주요 버그 수정
* **화면 이동 시 Toast 알림 중복 노출 해결**:
  - `activeScreen` 상태 변경(화면 전환) 시마다 대량의 리렌더링이 발생하며 "Supabase 클라우드 DB 연동 완료!" 메시지가 중복 및 반복 노출되는 문제를 해결하기 위해 `useRef` 플래그(`isFirstLoadRef`)와 `loadDatabase(true)` 인자를 통해 초기 최초 1회만 작동하도록 차단 처리.
* **카테고리별 테마 테일러링 색상 복원**:
  - Supabase 연동 모드일 때 대분류 아이콘 배경 및 텍스트 색상이 보라색으로 일원화되던 현상 수정. 카테고리 ID를 매핑하여 `cat-fatigue` (피로 개선 - 보라), `cat-diet` (다이어트 - 초록), `cat-hair` (탈모 - 파랑) 클래스가 동적으로 정상 부여되도록 수정.
* **쿠팡 상품 카드 클릭 시 즉시 리다이렉트**:
  - 상품 카드 클릭 시 발생하던 "쿠팡 파트너스 딥링크 리다이렉션..." 경고 팝업(`alert()`) 및 기본 동작 제한(`preventDefault()`)을 전면 제거하여, 클릭 즉시 새 창(`_blank`)으로 쿠팡 검색 링크로 부드럽게 이동하게 유도.

### 3) 쿠팡 파트너스 연동 및 중복 고지 제거
* **15종 성분별 고유 링크 매핑**: 15개 기능성 성분 데이터에 개별 쿠팡 파트너스 링크(`coupang_link` 필드)를 추가하고, 데이터 로드 시 이를 조회하여 딥링크로 우선적으로 연결하는 기능 지원.
* **중복 면책 고지 정리**: 결과 페이지의 개별 상품 카드 상단에 존재하던 쿠팡 파트너스 문구가 화면 최하단의 고정 스티키 배너와 중복 표출되던 문제를 발견하여, 상단 문구를 깔끔하게 정돈하고 하단 1회 단일 노출로 변경.

### 4) 식약처 고시 모달(KFDA Report Modal) 동적 필터링 적용
* **이슈**: 기존 KFDA Report 모달이 15개 전체 성분을 하드코딩으로 모두 보여주는 구조여서 사용자가 선택한 항목만 확인하기 어려웠음.
* **조치**: 
  - `App.jsx`에서 모달 컴포넌트로 현재 매칭된 성분 ID 목록(`matchedIngredientIds`)을 props로 전달하도록 연동.
  - 모달 내부에서 `matchedIngredientIds`에 포함된 항목만 필터링하여 노출하는 `filteredKfdaData` 로직을 구축.
  - 이에 따라 1개 증상 선택 시 해당 성분 1개만, 2개 선택 시 2개만 카테고리 구조를 보존한 채 깔끔하게 필터링되어 출력되도록 개선.

### 5) 법적 리스크 분석 및 컴플라이언스 준수 강화
* **"식약처 공인" 표현 삭제**: 서비스 매칭 로직이나 어플리케이션 자체에 부착 시 식약처 공인 사칭 리스크가 있었던 문구들을 전부 **"식약처 고시 데이터 참조/기반"** 등의 준수 표현으로 수정 완료.
* **개인 건강정보 수집 관련 고지 추가**: 사용자가 선택하는 설문 체크박스 하단에 *"위 정보는 브라우저 내에서만 처리되며 서버에 저장·전송되지 않습니다. 기저질환이 있는 경우 섭취 전 반드시 전문의와 상담하시기 바랍니다."* 문구를 표기하여 개인정보 보호 및 임의적 의료 권고 해석 방지.

* **법적 면책 조항 강화**: 모달 대시보드와 footer 영역에 *"본 서비스는 의료 행위가 아니며 전문가 상담을 대체하지 않습니다. 식약처가 매칭 로직을 승인한 사실이 없습니다"* 등의 문구를 삽입하여 의료법 및 표시광고법 위반 리스크 원천 방지.
